JSX Syntax & React Basics

⏱ 45 minutes  ·  Setup, JSX, expressions, and the rules of the template language

What is React?

React is a JavaScript library for building user interfaces. Created by Facebook (Meta) in 2013, it is today the most widely used frontend library in the world. React lets you break your UI into small, reusable pieces called components, and manages how those pieces update when your data changes.

💡

Core idea: In traditional web development, you manually update the DOM. React introduces a declarative model — you describe what the UI should look like for a given state, and React handles how to update the DOM efficiently.

Why learn React?

React powers millions of websites including Facebook, Instagram, Airbnb, Netflix, and WhatsApp Web. It has a massive ecosystem, strong job market demand, and once you understand its concepts, learning React Native (mobile) becomes much easier.

Introduction to JSX

JSX stands for JavaScript XML. It is a syntax extension that lets you write HTML-like markup directly inside JavaScript. JSX is not valid JavaScript on its own — a tool called Babel transforms it into regular JavaScript function calls before the browser runs it.

What you write (JSX) jsx 
const element = (
  <h1 className="greeting">
    Hello, World!
  </h1>
);
What Babel transforms it to (plain JS) js 
const element = React.createElement(
  "h1",
  { className: "greeting" },
  "Hello, World!"
);

This is why JSX is called syntactic sugar. You don't have to use JSX in React, but everyone does because it is far more readable than writing React.createElement calls manually.

The Rules of JSX

JSX looks like HTML but has a few important differences and rules you must follow.

Rule 1: Return a single root element

A component must return one top-level element. If you need to return multiple elements, wrap them in a <div> or a React Fragment.

jsx-rules.jsxjsx
// ❌ Wrong — two root elements
return (
  <h1>Title</h1>
  <p>Paragraph</p>
);

// ✅ Correct — wrapped in a Fragment
return (
  <>
    <h1>Title</h1>
    <p>Paragraph</p>
  </>
);
✅

React Fragment (<> </>) is an empty wrapper that groups elements without adding an extra DOM node. Use it instead of wrapping in an unnecessary <div>.

Rule 2: Close all tags

Every tag must be closed in JSX — including self-closing tags like <br />, <img />, and <input />.

jsx-tags.jsxjsx
// ❌ Wrong in JSX (valid HTML, but not JSX)
<img src="photo.jpg">
<input type="text">

// ✅ Correct in JSX
<img src="photo.jpg" />
<input type="text" />

Rule 3: camelCase attributes

HTML attribute names are written in camelCase in JSX. Two of the most common:

HTMLJSXWhy?
classclassNameclass is a reserved word in JS
forhtmlForfor is a reserved word in JS (loops)
onclickonClickJSX uses camelCase for events
tabindextabIndexStandard camelCase rule

Expressions in JSX

One of the most powerful features of JSX is the ability to embed any JavaScript expression inside your markup using curly braces { }.

Greeting.jsxjsx
function Greeting() {
  const name = "Amara";
  const age = 25;
  const isStudent = true;

  return (
    <div>
      {/* String variable */}
      <h1>Hello, {name}!</h1>

      {/* Math expression */}
      <p>In 5 years you will be {age + 5}</p>

      {/* Ternary (conditional) expression */}
      <p>{isStudent ? "You are a student" : "You are not a student"}</p>

      {/* Function call */}
      <p>Uppercase: {name.toUpperCase()}</p>
    </div>
  );
}
⚠️

Expressions only, not statements. You can use {name}, {2 + 2}, or {condition ? "yes" : "no"} inside JSX. You cannot put if/else statements or for loops directly inside { } — those are statements, not expressions.

Function Components

Modern React uses function components — plain JavaScript functions that return JSX. The function name must start with a capital letter. This is how React distinguishes your components from native HTML tags.

WelcomeCard.jsxjsx
// A simple function component
function WelcomeCard() {
  return (
    <div className="card">
      <h2>Welcome to React!</h2>
      <p>You are building your first component.</p>
    </div>
  );
}

// Export it so other files can import it
export default WelcomeCard;
ℹ️

Capital letter = component. Lowercase = HTML tag. When React sees <WelcomeCard />, it renders your function. When it sees <div>, it renders a native DOM element.

Arrow function syntax (alternative)

Many developers prefer arrow function syntax — both are valid:

Badge.jsxjsx
// Arrow function component
const Badge = () => {
  return <span className="badge">New!</span>;
};

// Or even shorter with implicit return:
const Badge = () => <span className="badge">New!</span>;

export default Badge;

Rendering Components

To use a component, you import it and use it like an HTML tag. The main.jsx entry point is where React attaches everything to your actual HTML page.

src/main.jsxjsx
import React from 'react';
import ReactDOM from 'react-dom/client';
import App from './App';

// Find the <div id="root"> in index.html
// and mount our React app inside it
ReactDOM
  .createRoot(document.getElementById('root'))
  .render(<App />);
src/App.jsxjsx
import WelcomeCard from './WelcomeCard';

function App() {
  return (
    <div>
      {/* Use your component like an HTML tag */}
      <WelcomeCard />
      <WelcomeCard />
      <WelcomeCard />
    </div>
  );
}

export default App;

Nesting Components

Components can be composed together — you can use a component inside another component. This is fundamental to building complex UIs from simple building blocks.

ProductCard.jsxjsx
// Small reusable sub-component
function StarRating() {
  return <span>⭐⭐⭐⭐⭐</span>;
}

function PriceTag() {
  return <strong className="price">₦4,500</strong>;
}

// Parent component using the above
function ProductCard() {
  return (
    <div className="product-card">
      <h3>Wireless Headphones</h3>
      <StarRating />
      <PriceTag />
      <button>Add to Cart</button>
    </div>
  );
}

export default ProductCard;
⚠️

Never define a component inside another component. Always define components at the top level of a file. Defining inside causes React to re-create the component on every render, breaking performance and state.

Rendering Lists

To render a list of items from an array, use the JavaScript .map() method inside JSX. Each item in the list must have a unique key prop so React can track which items changed.

FruitList.jsxjsx
function FruitList() {
  const fruits = ["Mango", "Banana", "Pawpaw", "Pineapple"];

  return (
    <ul>
      {fruits.map((fruit, index) => (
        <li key={index}>{fruit}</li>
      ))}
    </ul>
  );
}

// Better: use unique IDs as keys
function UserList() {
  const users = [
    { id: 1, name: "Chidi" },
    { id: 2, name: "Ngozi" },
    { id: 3, name: "Emeka" },
  ];

  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}

Props

⏱ 45 minutes  ·  Passing data into components to make them dynamic and reusable

Understanding Props

Props (short for properties) are how you pass data from a parent component to a child component. They make components reusable — instead of hardcoding "Mango" into your fruit card, you pass the name as a prop and reuse the same card component for any fruit.

💡

Props are read-only. A child component can read its props but must never modify them. This is the one-way data flow principle: data flows down from parent to child, never up.

Passing Props

You pass props to a component like HTML attributes. The child receives them all as a single object — conventionally called props.

App.jsx — the parentjsx
import ProductCard from './ProductCard';

function App() {
  return (
    <div>
      {/* Pass strings, numbers, and booleans as props */}
      <ProductCard
        name="Wireless Headphones"
        price={4500}
        inStock={true}
        rating={4.5}
      />
      <ProductCard
        name="Bluetooth Speaker"
        price={8900}
        inStock={false}
        rating={3.8}
      />
    </div>
  );
}
ProductCard.jsx — the childjsx
// props is an object containing all passed values
function ProductCard(props) {
  return (
    <div className="card">
      <h3>{props.name}</h3>
      <p>Price: ₦{props.price.toLocaleString()}</p>
      <p>Rating: {props.rating} / 5</p>
      <p>
        {props.inStock ? <span style={{color: "green"}}>In Stock</span> : <span style={{color: "red"}}>Sold Out</span>}
      </p>
    </div>
  );
}

export default ProductCard;

Destructuring Props

Writing props.name, props.price etc. everywhere gets verbose. Use destructuring in the function parameter to extract the values directly:

ProductCard.jsx — with destructuringjsx
// Destructure props directly in the parameter
function ProductCard({ name, price, inStock, rating }) {
  return (
    <div className="card">
      <h3>{name}</h3>
      <p>Price: ₦{price.toLocaleString()}</p>
      <p>Rating: {rating} / 5</p>
      <p>{inStock ? "✅ In Stock" : "❌ Sold Out"}</p>
    </div>
  );
}

export default ProductCard;
✅

Best practice: Always destructure props. It makes your component signature a clear declaration of what data it needs — like a contract. Any reader knows immediately that ProductCard expects name, price, inStock, and rating.

Default Props

You can provide default values for props that might not be passed. This makes your components more robust and prevents errors when a prop is undefined.

function Avatar({

State Management

⏱ 40 minutes  ·  Making components interactive with React's useState hook

What is State?

While props are data passed into a component, state is data that lives inside a component and can change over time. When state changes, React automatically re-renders the component to reflect the new data.

💡

Mental model: Think of a light switch. The switch's position (on/off) is state. When you flip it, the component re-renders — the light turns on or off. React "reacts" to state changes.

PropsState
Where it comes fromPassed by parentLives inside the component
Can component change it?No (read-only)Yes (that's the point!)
When does UI update?When parent re-rendersWhen setState is called
Use caseConfiguration, dataUser interactions, dynamic data

The useState Hook

State in function components is managed using the useState Hook. A hook is a special function provided by React. useState returns an array with two items: the current value, and a function to update it.

Counter.jsxjsx
import { useState } from 'react';

function Counter() {
  // Declare state: [currentValue, setterFunction]
  // The argument to useState() is the INITIAL value
  const [count, setCount] = useState(0);

  return (
    <div>
      <h2>Count: {count}</h2>

      {/* Call setCount to update state */}
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>

      <button onClick={() => setCount(count - 1)}>
        Decrement
      </button>

      <button onClick={() => setCount(0)}>
        Reset
      </button>
    </div>
  );
}

export default Counter;
ℹ️

useState naming convention: Always name them [something, setSomething]. For example: [isOpen, setIsOpen], [username, setUsername], [items, setItems]. This is the universal React convention.

State can hold any data type

state-types.jsxjsx
// Number
const [score, setScore] = useState(0);

// String
const [name, setName] = useState("");

// Boolean (great for toggles)
const [isVisible, setIsVisible] = useState(true);

// Array
const [items, setItems] = useState([]);

// Object
const [user, setUser] = useState({ name: "", age: 0 });

Updating State

When updating state based on the previous value, use the functional update form — pass a function to setState instead of a value. This guarantees you're working with the most recent state.

SafeCounter.jsxjsx
// ⚠️ Can cause bugs if updates are batched
setCount(count + 1);

// ✅ Safe — always uses the latest state
setCount(prevCount => prevCount + 1);

Updating objects and arrays in state

React state must be treated as immutable. Never mutate objects or arrays directly — always create a new copy with your changes.

state-arrays.jsxjsx
const [items, setItems] = useState(["Apple", "Banana"]);

// ❌ WRONG — mutating state directly
items.push("Cherry");
setItems(items);

// ✅ CORRECT — create a new array
setItems([...items, "Cherry"]);

// Remove an item
setItems(items.filter(item => item !== "Banana"));

// Update an object in state
const [user, setUser] = useState({ name: "Temi", age: 22 });

// ✅ Spread to copy, then override the field
setUser({ ...user, age: 23 });

A toggle example

ToggleButton.jsxjsx
import { useState } from 'react';

function ToggleButton() {
  const [isOn, setIsOn] = useState(false);

  return (
    <button
      onClick={() => setIsOn(prev => !prev)}
      style={{ background: isOn ? "green" : "gray" }}
    >
      {isOn ? "ON 🟢" : "OFF 🔴"}
    </button>

}

Forms and State

Forms are where state truly shines. Controlled components link form inputs to React state — the input's value is always driven by state, and every keystroke updates the state. This gives you full control over the form data.

LoginForm.jsxjsx
import { useState } from 'react';

function LoginForm() {
  const [email, setEmail] = useState("");
  const [password, setPassword] = useState("");
  const [message, setMessage] = useState("");

  function handleSubmit(e) {
    e.preventDefault(); // Stop page reload

    if (email && password) {
      setMessage(`Welcome, ${email}!`);
    } else {
      setMessage("Please fill in all fields.");
    }
  }

  return (
    <form onSubmit={handleSubmit}>
      <div>
        <label>Email:</label>
        <input
          type="email"
          value={email}
          onChange={(e) => setEmail(e.target.value)}
        />
      </div>
      <div>
        <label>Password:</label>
        <input
          type="password"
          value={password}
          onChange={(e) => setPassword(e.target.value)}
        />
      </div>
      <button type="submit">Login</button>
      {message && <p>{message}</p>}
    </form>
  );
}

Lifting State Up

When two sibling components need to share the same state, lift the state up to their nearest common parent. The parent holds the state and passes both the value and the update function down as props.

lifting-state.jsxjsx
import { useState } from 'react';

// Child 1: Displays the score
function ScoreDisplay({ score }) {
  return <h2>Score: {score}</h2>;
}

// Child 2: Controls the score
function ScoreControls({ onAdd, onReset }) {
  return (
    <div>
      <button onClick={onAdd}>+10 Points</button>
      <button onClick={onReset}>Reset</button>
    </div>
  );
}

// Parent: owns the state, passes it down
function ScoreBoard() {
  const [score, setScore] = useState(0);

  return (
    <div>
      <ScoreDisplay score={score} />
      <ScoreControls
        onAdd={() => setScore(prev => prev + 10)}
        onReset={() => setScore(0)}
      />
    </div>
  );
}
💡

The golden rule: State lives as low as possible in the component tree, but as high as necessary for sharing. Start with state in the component that needs it. If a sibling also needs it, move it up to the parent.

useEffect Hook

⏱ 50 minutes  ·  Side effects, data fetching, timers, and cleanup

What is useEffect?

A side effect is anything a component does that reaches outside React's rendering system — fetching data from an API, setting a timer, manually updating the document title, or subscribing to a data stream. The useEffect hook is where you safely run these operations.

💡

Mental model: React renders your component (runs the function, paints the UI). After each render, React checks if any effects need to run and calls them. Effects are always scheduled after the DOM has been updated.

basic-effect.jsxjsx
import { useState, useEffect } from 'react';

function DocumentTitleUpdater() {
  const [count, setCount] = useState(0);

  // This runs after EVERY render
  useEffect(() => {
    document.title = `You clicked ${count} times`;
  });

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(c => c + 1)}>Click me</button>
    </div>
  );
}

The basic signature of useEffect is:

useEffect signaturejsx
useEffect(function effect() {
  // your side effect here

  return function cleanup() { // optional
    // undo the side effect
  };
}, [/* dependency array */]);

The Dependency Array

The second argument to useEffect is the dependency array — it controls when the effect re-runs. This is the most important thing to understand about useEffect.

Dependency ArrayWhen the Effect Runs
useEffect(fn) — omittedAfter every render
useEffect(fn, []) — empty arrayOnly once, after the first render (mount)
useEffect(fn, [a, b]) — with valuesAfter first render, and whenever a or b changes
dependency-examples.jsxjsx
// 1. Run once on mount — good for initialization
useEffect(() => {
  console.log("Component mounted!");
}, []); // empty array = run once

// 2. Run when 'userId' changes — good for re-fetching
useEffect(() => {
  console.log(`User changed to: ${userId}`);
}, [userId]);

// 3. Run when either value changes
useEffect(() => {
  console.log("page or search changed");
}, [page, searchQuery]);
⚠️

ESLint will warn you if you use a variable inside useEffect without listing it in the dependency array. This is usually the right warning — always include every reactive value your effect uses.

Fetching Data with useEffect

The most common use of useEffect is fetching data from an API when a component mounts. The pattern is: track loading/error/data in state, and trigger the fetch in an effect.

UserProfile.jsxjsx
import { useState, useEffect } from 'react';

function UserProfile({ userId }) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    // Reset state when userId changes
    setLoading(true);
    setError(null);

    fetch(`https://jsonplaceholder.typicode.com/users/${userId}`)
      .then(res => {
        if (!res.ok) throw new Error("User not found");
        return res.json();
      })
      .then(data => {
        setUser(data);
        setLoading(false);
      })
      .catch(err => {
        setError(err.message);
        setLoading(false);
      });
  }, [userId]); // re-run whenever userId prop changes

  if (loading) return <p>Loading...</p>;
  if (error)   return <p style={{color:"red"}}>Error: {error}</p>;

  return (
    <div>
      <h2>{user.name}</h2>
      <p>Email: {user.email}</p>
      <p>City: {user.address.city}</p>
    </div>
  );
}

export default UserProfile;
✅

Always handle loading and error states. Real APIs can be slow or fail. Your UI should communicate both situations clearly rather than showing a blank screen.

Cleanup Functions

When a component is removed from the screen (unmounts), or before an effect re-runs, you need to clean up any subscriptions, timers, or listeners — otherwise you get memory leaks. Return a cleanup function from your effect.

TimerComponent.jsxjsx
import { useState, useEffect } from 'react';

function LiveClock() {
  const [time, setTime] = useState(new Date());

  useEffect(() => {
    // Start a timer
    const intervalId = setInterval(() => {
      setTime(new Date());
    }, 1000);

    // Cleanup: stop the timer when component unmounts
    return () => clearInterval(intervalId);
  }, []); // run once

  return <h2>🕐 {time.toLocaleTimeString()}</h2>;
}

// Cleanup also prevents stale state on async fetches
function SearchResults({ query }) {
  const [results, setResults] = useState([]);

  useEffect(() => {
    let cancelled = false; // cleanup flag

    fetch(`/api/search?q=${query}`)
      .then(r => r.json())
      .then(data => {
        if (!cancelled) setResults(data);
      });

    return () => { cancelled = true; }; // cancel stale response
  }, [query]);

  return <ul>{results.map(r => <li key={r.id}>{r.name}</li>)}</ul>;
}

useRef Hook

⏱ 35 minutes  ·  Direct DOM access, focus management, and persisting values across renders

What is useRef?

useRef returns a mutable object { current: initialValue }. Unlike state, changing ref.current does NOT trigger a re-render. This makes it ideal for two specific use cases: accessing DOM elements directly, and storing values that need to persist across renders without causing re-renders.

💡

Two jobs of useRef: (1) Hold a reference to a real DOM node so you can call methods like .focus() or .scrollIntoView(). (2) Store any mutable value — like a timer ID or previous state value — that you need to remember but don't want to trigger re-renders.

useRef anatomyjsx
import { useRef } from 'react';

function MyComponent() {
  // Create a ref — starts as null for DOM refs
  const myRef = useRef(null);

  // Attach to a DOM element with the ref prop
  return <input ref={myRef} />;

  // After render, myRef.current === the <input> DOM node
}

DOM Access with useRef

The most common use: auto-focusing an input when a component mounts, or managing focus in complex UI like modals and dropdowns.

AutoFocusInput.jsxjsx
import { useRef, useEffect } from 'react';

function SearchInput() {
  const inputRef = useRef(null);

  // Auto-focus the input on mount
  useEffect(() => {
    inputRef.current.focus();
  }, []);

  return (
    <div>
      <input ref={inputRef} placeholder="Search..." />
      <button onClick={() => inputRef.current.focus()}>
        Focus Input
      </button>
    </div>
  );
}

// Video player control
function VideoPlayer({ src }) {
  const videoRef = useRef(null);

  return (
    <div>
      <video ref={videoRef} src={src} />
      <button onClick={() => videoRef.current.play()}>Play</button>
      <button onClick={() => videoRef.current.pause()}>Pause</button>

Persisting Values Across Renders

The second use case: storing a value that should survive re-renders without causing them. A classic example is storing a timer ID so you can cancel it later.

StopwatchWithRef.jsxjsx
import { useState, useRef } from 'react';

function Stopwatch() {
  const [seconds, setSeconds] = useState(0);
  const [running, setRunning] = useState(false);

  // Store timer ID — changing this shouldn't re-render
  const intervalRef = useRef(null);

  function start() {
    if (running) return;
    setRunning(true);
    intervalRef.current = setInterval(() => {
      setSeconds(s => s + 1);
    }, 1000);
  }

  function stop() {
    clearInterval(intervalRef.current);
    setRunning(false);
  }

  function reset() {
    stop();
    setSeconds(0);
  }

  return (
    <div>
      <h2>{seconds}s</h2>
      <button onClick={start}>Start</button>
      <button onClick={stop}>Stop</button>
      <button onClick={reset}>Reset</button>
    </div>
  );
}

Event Handling in React

⏱ 40 minutes  ·  Handling user interactions — clicks, inputs, keyboard, forms, and more

Event Handling Basics

React handles events using camelCase event handlers directly on JSX elements. Instead of onclick="doSomething()" in HTML, you pass a function reference to onClick in JSX.

basic-events.jsxjsx
function ButtonDemo() {
  // Define the handler function
  function handleClick() {
    alert("Button was clicked!");
  }

  // ✅ Pass the function reference — no parentheses!
  return <button onClick={handleClick}>Click Me</button>;

  // ❌ Wrong: this CALLS the function immediately on render
  return <button onClick={handleClick()}>Click Me</button>;
}
⚠️

Never call handlers with () directly in JSX. Writing onClick={handleClick()} executes the function immediately during render — instead write onClick={handleClick} (pass the reference) or onClick={() => handleClick()} (wrap in an arrow function).

Synthetic Events and the Event Object

React wraps native browser events in a SyntheticEvent object — a cross-browser wrapper that ensures events behave consistently in all browsers. Event handlers receive this object as their first argument.

event-object.jsxjsx
function EventDemo() {
  // The event object 'e' is passed automatically
  function handleChange(e) {
    console.log(e.target.value);  // current input text
    console.log(e.target.name);   // input's name attribute
    console.log(e.target.checked); // checkbox state
  }

  function handleSubmit(e) {
    e.preventDefault(); // stop page reload!
    console.log("Form submitted");
  }

  function handleKeyDown(e) {
    if (e.key === "Enter") {
      console.log("Enter pressed!");
    }
    if (e.key === "Escape") {
      console.log("Closing...");
    }
  }

  return (
    <form onSubmit={handleSubmit}>
      <input
        onChange={handleChange}
        onKeyDown={handleKeyDown}
        placeholder="Type something"
      />
    </form>
  );
}

if / else in JSX

Because JSX is JavaScript, you can use standard if/else logic before the return statement to decide what to render. This is the clearest and most readable approach for complex conditions.

conditional-if.jsxjsx
function Dashboard({ isLoggedIn, role }) {
  // Early return pattern — clean and readable

    return <p>Please log in to continue.</p>;

    return <AdminPanel />;
  }

  return <UserDashboard />;

// Or use a variable to hold JSX
function StatusBadge({ status }) {
  let badge;

  if (status === "active") {
    badge = <span style={{color: "green"}}>● Active</span>;
  } else if (status === "pending") {
    badge = <span style={{color: "orange"}}>● Pending</span>;
  } else {
    badge = <span style={{color: "gray"}}>● Inactive</span>;
  }

  return <div>Status: {badge}</div>;

Ternary and Short-circuit Rendering

For inline conditions inside JSX, two patterns are standard: the ternary operator (condition ? A : B) for showing one of two things, and the && operator for showing something or nothing.

inline-conditions.jsxjsx
function NotificationBell({ count, isPremium, errorMessage }) {
  return (
    <div>
      {/* Ternary: show one of two things */}
      {count > 0
        ? <span className="badge">{count}</span>
        : <span>No notifications</span>
      }

      {/* && short-circuit: show or show nothing */}
      {isPremium && <span className="crown">👑 Premium</span>}

      {/* Common gotcha: use !! or Boolean() for numbers */}
      {!!count && <span>You have messages</span>}
      {/* Without !!, if count=0 React renders "0" to the DOM */}

      {/* Nullish coalescing for fallbacks */}
      {errorMessage && <p className="error">{errorMessage}</p>}
    </div>
  );
}
⚠️

The 0 gotcha: {count && <Component />} will render the number 0 to the DOM if count is 0, because 0 is falsy but React renders it. Use {count > 0 && ...} or {!!count && ...} to be safe.

Advanced List Rendering

Beyond basic .map(), you'll frequently combine filtering, sorting, and conditional rendering when working with lists of real data.

advanced-lists.jsxjsx
import { useState } from 'react';

const EMPLOYEES = [
  { id: 1, name: "Amaka",    dept: "Engineering", active: true  },
  { id: 2, name: "Tunde",    dept: "Design",      active: true  },
  { id: 3, name: "Chidera",  dept: "Engineering", active: false },
  { id: 4, name: "Funke",    dept: "Marketing",   active: true  },
  { id: 5, name: "Emeka",    dept: "Design",      active: false },
];

function EmployeeDirectory() {
  const [search, setSearch] = useState("");
  const [dept, setDept] = useState("All");
  const [showActive, setShowActive] = useState(false);

  // Chain filter operations — derived from state
  const filtered = EMPLOYEES
    .filter(e => dept === "All" || e.dept === dept)
    .filter(e => !showActive || e.active)
    .filter(e => e.name.toLowerCase().includes(search.toLowerCase()));

  return (
    <div>
      {/* Filter controls */}
      <input value={search} onChange={e => setSearch(e.target.value)} placeholder="Search..." />
      <select value={dept} onChange={e => setDept(e.target.value)}>
        {["All", "Engineering", "Design", "Marketing"].map(d => (
          <option key={d}>{d}</option>
        ))}
      </select>
      <label>
        <input type="checkbox" checked={showActive} onChange={e => setShowActive(e.target.checked)} />
        Active only
      </label>

      {/* Empty state */}
      {filtered.length === 0
        ? <p>No employees match your filters.</p>
        : (
          <ul>
            {filtered.map(emp => (
              <li key={emp.id}>
                {emp.name} — {emp.dept}
                {emp.active ? <span style={{color:"green"}}> ✓</span> : <span style={{color:"gray"}}> ✗</span>}
              </li>
            ))}
          </ul>
        )
      }
    </div>
  );
}
✅

Always handle the empty state. When a filtered list is empty, show a helpful message like "No results found" rather than a blank area. Users need to know their filter worked — there's just nothing matching.

Composition Patterns

React favours composition over inheritance. Rather than extending base classes, you build complex components by combining simpler ones. This section explores the patterns that make components truly reusable across a project.

💡

The key question to ask: "Could I drop this component into a different project without changes?" If the answer is no, the component probably has hardcoded assumptions that should be replaced with props or composition.

The Slot Pattern

Named "slots" let you pass multiple distinct pieces of JSX to different areas of a layout component — similar to how HTML <slot> works in Web Components. You do this with custom props.

// Layout component with header, body, and footer slots
function Modal({ header, children, footer, onClose }) {
  return (

        <div className="modal-body">{children}</div>

        {/* Footer slot */}

      </div>
    </div>
  );
}

Custom Hooks

A custom hook is a regular JavaScript function whose name starts with use, and that calls other React hooks inside. Custom hooks let you extract and share stateful logic — not UI — between components. They are the most powerful reusability tool in React.

hooks/useLocalStorage.jsjsx
import { useState } from 'react';

// Custom hook: persists state in localStorage
function useLocalStorage(key, initialValue) {
  const [storedValue, setStoredValue] = useState(() => {
    try {
      const item = window.localStorage.getItem(key);
      return item ? JSON.parse(item) : initialValue;
    } catch {
      return initialValue;
    }
  });

  const setValue = (value) => {
    setStoredValue(value);
    window.localStorage.setItem(key, JSON.stringify(value));
  };

  return [storedValue, setValue];
}

// hooks/useFetch.js — custom hook for data fetching
function useFetch(url) {
  const [data, setData]       = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError]     = useState(null);

  useEffect(() => {
    fetch(url)
      .then(r => r.json())
      .then(d => { setData(d); setLoading(false); })
      .catch(e => { setError(e.message); setLoading(false); });
  }, [url]);

  return { data, loading, error };
}

// Both hooks used together in a component
function UserSettings() {
  const [theme, setTheme] = useLocalStorage("theme", "dark");
  const { data: user, loading } = useFetch("/api/me");

  if (loading) return <p>Loading...</p>;

  return (
    <div>
      <h2>Welcome, {user.name}</h2>
      <button onClick={() => setTheme(t => t === "dark" ? "light" : "dark")}>
        Switch to {theme === "dark" ? "Light" : "Dark"} Mode
      </button>
    </div>
  );
}
✅

Custom hooks are just functions. They follow the same rules as hooks (only call them at the top level of a component or another hook), but they're purely JavaScript. They return whatever is useful — values, functions, or an object of both. Name them starting with use so React's linting rules apply.

import { useState } from 'react';

// 1. TodoItem component (props: text, done, onToggle, onDelete)
function TodoItem({ text, done, onToggle, onDelete }) {
  return (
    <li style={{
      display: "flex", alignItems: "center", gap: "8px",
      textDecoration: done ? "line-through" : "none",
      color: done ? "#888" : "inherit"
    }}>

        onChange={onToggle}
      />

      <button onClick={onDelete}>🗑</button>
    </li>

// 2. AddTodoForm component (props: onAdd)
function AddTodoForm({ onAdd }) {
  const [text, setText] = useState("");

  function handleSubmit(e) {
    e.preventDefault();
    if (!text.trim()) return;
    onAdd(text);
    setText(""); // clear input after adding
  }

  return (
    <form onSubmit={handleSubmit} style={{display: "flex", gap: "8px"}}>
      <input
        value={text}
        onChange={e => setText(e.target.value)}
        placeholder="Add a new task..."
        style={{flex: 1, padding: "8px"}}
      />
      <button type="submit">Add</button>
    </form>
  );
}

// 3. Main TodoApp — holds all state
function TodoApp() {
  const [todos, setTodos] = useState([
    { id: 1, text: "Learn React JSX", done: true },
    { id: 2, text: "Understand Components", done: true },
    { id: 3, text: "Master Props", done: false },
    { id: 4, text: "Learn State", done: false },
  ]);

  function addTodo(text) {
    const newTodo = { id: Date.now(), text, done: false };
    setTodos(prev => [...prev, newTodo]);
  }

  function toggleTodo(id) {
    setTodos(prev =>
      prev.map(todo =>
        todo.id === id ? { ...todo, done: !todo.done } : todo
      )
    );
  }

  function deleteTodo(id) {
    setTodos(prev => prev.filter(todo => todo.id !== id));
  }

  const completedCount = todos.filter(t => t.done).length;

  return (
    <div style={{maxWidth: "500px", margin: "40px auto", padding: "24px"}}>
      <h1>My Todo List</h1>
      <p>{completedCount} / {todos.length} completed</p>

      <AddTodoForm onAdd={addTodo} />

      <ul style={{marginTop: "16px", padding: 0, listStyle: "none"}}>
        {todos.map(todo => (
          <TodoItem
            key={todo.id}
            text={todo.text}
            done={todo.done}
            onToggle={() => toggleTodo(todo.id)}
            onDelete={() => deleteTodo(todo.id)}
          />
        ))}
      </ul>
    </div>
  );
}

export default TodoApp;
